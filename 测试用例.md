# 电商风控系统 — 验收测试用例

## 测试环境

- **Base URL:** `http://localhost:8000`
- **认证方式:** Bearer Token（通过 `/api/auth/login` 获取）
- **测试账号:** `admin / admin123`

---

## 一、四类业务事件风险检查

> **验收标准：** 能发起四类业务事件的风险检查

### TC-1.1 订单创建事件（order_create）

**前置条件：** 用户已登录，获取 Token

**请求：**

```http
POST /api/risk/check
Authorization: Bearer {token}
Content-Type: application/json

{
  "event_type": "order_create",
  "source_id": "tc1.1-order-001",
  "user_id": "U_TEST_001",
  "order_id": "O_TEST_001",
  "event_payload": {
    "order_amount": 5500,
    "order_item_count": 3,
    "user_register_days": 3,
    "payment_method": "alipay"
  }
}
```

**预期结果：**

| 字段 | 预期值 | 说明 |
|------|--------|------|
| code | 0 | 请求成功 |
| data.event_type | `order_create` | 事件类型正确 |
| data.risk_score | ≥ 0 | 评分已计算 |
| data.risk_level | `low` / `medium` / `high` | 等级已生成 |
| data.decision | `pass` / `manual_review` / `reject` | 处理建议已生成 |
| data.rule_hits | 数组 | 命中规则列表 |
| data.feature_snapshot | 对象 | 特征快照 |

## 三、高风险自动创建案件

> **验收标准：** 高风险结果能自动创建案件

### TC-3.1 高风险事件自动生成案件

**步骤 1：** 发起高风险检查

```http
POST /api/risk/check
Authorization: Bearer {token}
Content-Type: application/json

{
  "event_type": "order_create",
  "source_id": "tc3.1-case-001",
  "user_id": "U_CASE_001",
  "order_id": "O_CASE_001",
  "event_payload": {
    "order_amount": 12000,
    "order_item_count": 25,
    "user_register_days": 1,
    "device_related_user_count": 10,
    "is_ip_high_risk_area": true,
    "payment_method": "virtual_card"
  }
}
```

**预期结果：**

| 字段 | 预期值 | 说明 |
|------|--------|------|
| data.risk_level | `high` | 高风险 |
| data.case_id | 非空字符串 | 案件已创建 |

**步骤 2：** 查询案件详情

```http
GET /api/cases/{case_id}
Authorization: Bearer {token}
```

**预期结果：**

| 字段 | 预期值 | 说明 |
|------|--------|------|
| code | 0 | 请求成功 |
| data.id | 与 case_id 一致 | 案件 ID 匹配 |
| data.user_id | `U_CASE_001` | 用户 ID 正确 |
| data.risk_score | ≥ 70 | 风险分数正确 |
| data.risk_level | `high` | 风险等级正确 |
| data.status | `pending_review` | 待审核状态 |
| data.event_type | `order_create` | 事件类型正确 |

## 四、案件审核与审核日志

> **验收标准：** 能在案件页完成审核并查看审核日志

### TC-4.1 审核通过

**前置条件：** TC-3.1 已执行，案件已创建

**步骤 1：** 获取待审核案件列表

```http
GET /api/cases?status=pending_review
Authorization: Bearer {token}
```

**预期结果：**

| 字段 | 预期值 | 说明 |
|------|--------|------|
| code | 0 | 请求成功 |
| data.items | 数组 | 包含待审核案件 |
| data.items[0].status | `pending_review` | 状态正确 |

**步骤 2：** 执行审核通过

```http
POST /api/cases/{case_id}/review
Authorization: Bearer {token}
Content-Type: application/json

{
  "action": "approve",
  "remark": "核实后确认为正常订单，审核通过"
}
```

**预期结果：**

| 字段 | 预期值 | 说明 |
|------|--------|------|
| code | 0 | 审核成功 |
| data.status | `approved` | 状态变更为已通过 |

**步骤 3：** 验证案件状态已更新

```http
GET /api/cases/{case_id}
Authorization: Bearer {token}
```

**预期结果：**

| 字段 | 预期值 | 说明 |
|------|--------|------|
| data.status | `approved` | 状态已更新 |
| data.review_remark | 包含审核意见 | 审核备注已记录 |
| data.reviewed_at | 非空 | 审核时间已记录 |

---

### TC-4.2 审核拒绝

**前置条件：** TC-3.2 已执行，案件已创建

**请求：**

```http
POST /api/cases/{case_id}/review
Authorization: Bearer {token}
Content-Type: application/json

{
  "action": "reject",
  "remark": "经核实为恶意刷单，拒绝订单"
}
```

**预期结果：**

| 字段 | 预期值 | 说明 |
|------|--------|------|
| code | 0 | 审核成功 |
| data.status | `rejected` | 状态变更为已拒绝 |

---

### TC-4.3 查看审核日志

**前置条件：** TC-4.1 和 TC-4.2 已执行

**请求：**

```http
GET /api/cases/{case_id}/logs
Authorization: Bearer {token}
```

**预期结果：**

| 字段 | 预期值 | 说明 |
|------|--------|------|
| code | 0 | 请求成功 |

## 八、完整演示链路

> **验收标准：** 能完成一条完整演示链路

### TC-8.1 端到端完整链路

**场景：** 新用户大额下单 → 风险检查命中 → 自动创建案件 → 人工审核 → 结果回查

---

**步骤 1：发起风险检查**

```http
POST /api/risk/check
Authorization: Bearer {token}
Content-Type: application/json

{
  "event_type": "order_create",
  "source_id": "tc8.1-e2e-001",
  "user_id": "U_E2E_001",
  "order_id": "O_E2E_001",
  "event_payload": {
    "order_amount": 8500,
    "order_item_count": 15,
    "user_register_days": 2,
    "device_related_user_count": 6,
    "is_ip_high_risk_area": true,
    "payment_method": "virtual_card"
  }
}
```

**预期结果：**

| 字段 | 预期值 | 说明 |
|------|--------|------|
| data.risk_score | ≥ 60 | 中/高风险 |
| data.risk_level | `medium` 或 `high` | 需要审核 |
| data.decision | `manual_review` | 需人工审核 |
| data.case_id | 非空 | 案件已创建 |
| data.rule_hits | ≥ 3 条 | 多条规则命中 |
| data.feature_snapshot | 对象 | 特征快照完整 |

> **记录返回的 `case_id`，后续步骤使用。**

---

**步骤 2：查看案件详情**

```http
GET /api/cases/{case_id}
Authorization: Bearer {token}
```

**预期结果：**

| 字段 | 预期值 | 说明 |
|------|--------|------|
| data.id | 与 step1 返回的 case_id 一致 | 案件 ID 匹配 |
| data.user_id | `U_E2E_001` | 用户 ID 正确 |
| data.risk_score | 与 step1 一致 | 风险分数一致 |
| data.status | `pending_review` | 待审核状态 |
| data.rule_hits | 与 step1 一致 | 命中规则一致 |
| data.feature_snapshot | 与 step1 一致 | 特征快照一致 |

---

**步骤 3：查看案件关联的规则命中详情**

```http
GET /api/cases/{case_id}/rule-hits
Authorization: Bearer {token}
```

**预期结果：**

| 字段 | 预期值 | 说明 |
|------|--------|------|
| code | 0 | 请求成功 |
| data | 数组 | 命中规则列表 |
| data[*].rule_code | 非空 | 规则代码 |
| data[*].rule_name | 非空 | 规则名称 |
| data[*].hit_score | > 0 | 命中分数 |

---

**步骤 4：人工审核通过**
